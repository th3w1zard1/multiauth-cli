# MCP / IDE handoff (manual merge)

**multiauth-cli** pools credentials for **child processes** you start from a terminal. Tooling that is **MCP, extensions, or built-in IDE HTTP clients** use **their own** env and are **not** routed through a PATH shim.

To align behavior:

1. **Terminal / tasks:** use the **shim** (or `multiauth-run --profile <name> …`) and the same profile as in your `profiles.yaml` / TOML.
2. **MCP servers:** set the API key (or other secret) the **MCP process** reads in the MCP **JSON config** (Cursor: project or user `mcp.json` / app settings, depending on version). Use **multiple keys** or rotation **only** if the server supports that; a single `MULTIAUTH_*` pool in this package does not wrap an already-running MCP unless you add a custom launcher that injects one key at a time.

### Example: JSON fragment (illustration only)

Replace the placeholder and merge into your MCP config **manually**; do not commit real secrets.

```json
{
  "mcpServers": {
    "example-cli-host": {
      "command": "npx",
      "args": ["-y", "example-pkg", "mcp"],
      "env": {
        "SOME_API_KEY": "<single-key-or-your-policy>"
      }
    }
  }
}
```

3. Re-read **docs/HOSTS.md** in this repo for the shell vs host boundary.
